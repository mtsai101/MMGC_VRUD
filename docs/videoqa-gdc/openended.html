Introduction

In open-ended QA, no candidate answers are available as inputs, and the models are required to automatically generate the answer in natural language according to the given videos and questions. We report WUPS score for performance evaluation. Open-ended QA challenges not only visual reasoning but also language generation which is also an interesting research problem that can be hosted in VQA. Our preliminary experiments show that VideoQA models that are effective in multi-choice QA perform poorly in the open-ended scenario. Hopefully, this problem can be better solved by using pre-trained architectures.
